6942. 树中可以形成回文的路径数

位运算：

回文串等价于至多一个字母出现奇数次，其余字母出现偶数次（两个奇数次字符串无法满足对称性）。

用一个长为 26 的二进制数来压缩存储每个字母的奇偶性，一条边可以看成是 1<<(s[i]−′a′)1 << (s[i]-'a')1<<(s[i]−′a′)。

那么路径所对应的二进制数，就是路径上的所有边的异或和（因为异或就是摸 222 剩余系中的加法，刚好可以表示奇偶性）。

只有 27 个二进制数符合要求：

  • 000 表示每个字母都出现偶数次
  • 20,21,...2252^0,2^1,...2^{25}20,21,...225，表示第 iii 个字母出现奇数次，其余字母出现偶数次

设 vvv 和 www 的最近公共祖先为 lcalcalca，设从根到 xxx 的路径异或和为 XORvXOR_vXORv​。

vvv 到 www 的路径可以看出是 v−lca−wv - lca - wv−lca−w，其中 lcalcalca 到 vvv 的路径异或和，等于跟到 vvv 的异或和，再加上跟到 lcalcalca 的异或和。（从根到 lcalcalca 的边异或了两次，等于 0 低效掉。）

所以 v−lca−wv - lca - wv−lca−w 异或和为：

(XORv⊕XORlca)⊕(XORw⊕XORlca)(XOR_v \oplus XOR_{lca}) \oplus (XOR_w \oplus XOR_{lca}) (XORv​⊕XORlca​)⊕(XORw​⊕XORlca​)

XORlcaXOR_{lca}XORlca​ 异或了两次，抵消掉，所以上式为

XORv⊕XORwXOR_v \oplus XOR_w XORv​⊕XORw​

所有 XORXORXOR 求出来，就变成判断 n−1n-1n−1 个数当中：

  • 两数异或和是否为 000？这意味着路径上的每个字母都出现偶数次。
  • 两数异或和是否为 222 的幂？这意味着路径上恰好有个字母出现奇数次，其余字母为偶数次
  • 特殊情况：XORv=0XOR_v = 0XORv​=0 或者 $XOR_v $ 为 222 的幂，表示从根到 vvv 的路径满足要求，我们可以异或上一条「空路径」对应的异或值(000)，就转换为了上面两数异或和的情况。

这可以用类似两数之和的思路解决，用哈希表记录 XORvXOR_vXORv​ 的个数，设当前算出的异或和为 xxx，去哈希表中找 xxx 的出现次数以及 x⊕2ix \oplus 2^ix⊕2i 的出现次数
